## AI Assistant [18:05:20]

**Playwright** is a modern, powerful framework developed by Microsoft for end-to-end (E2E) web testing. Unlike unit tests or standard backend integration tests that simulate requests in memory, Playwright launches a real web browser (like Google Chrome, Firefox, or Safari), navigates to your website, and interacts with the page exactly as a human user would (clicking buttons, typing into fields, submitting forms, and asserting page content).

### Why did I write it in Python?
1. **No Node.js/NPM Installed**: I checked your system environment, and Node.js (`node`) is not installed. Usually, Playwright is run via JavaScript/Node.js, but since it was missing, a Node version wouldn't compile or run.
2. **Python is Available**: Your system has **Python 3.9.6** installed and ready. Playwright has official, fully-supported Python bindings.
3. **No .NET Project Pollution**: While you *can* run Playwright in C#, doing so requires adding NuGet packages and setting up browser executable targets inside MSBuild which can sometimes fail or slow down regular unit test builds. Writing it as a separate Python script makes it a lightweight, self-contained automated script that doesn't interfere with your core C# code.

### What does the script do?
The [playwright_scenario.py](file:///Users/jakovivancek/Desktop/Jakov_Ivancek_projekt/playwright_scenario.py) script automatically performs a **10-step E2E UI scenario** to fulfill the `+3 extra boda` checklist requirement:
1. Spawns your ASP.NET Core web server on a background port (`http://127.0.0.1:5078`).
2. Opens a headless Chromium browser window and navigates to the `/Account/Register` page.
3. Fills out the registration form with a new random test user and submits it.
4. Verifies the page displays the "Signed in" header.
5. Navigates to `/Setup` and promotes the new user to the **Admin** role.
6. Logs out of the current session.
7. Navigates to `/Account/Login` and signs back in with the promoted Admin user (to refresh claims).
8. Navigates to the `/hoteli/create` page.
9. Fills out name, city, address, rating, and submits the form to create a new Hotel.
10. Navigates to `/hoteli` index and asserts that the new hotel ("Playwright Grand Hotel") is visible in the list.
11. Terminates the web server process and closes the browser.

### How to run it:
If you or the evaluator want to run this test, you only need to run these commands in your terminal:
```bash
# 1. Install the Playwright package for Python
pip3 install playwright

# 2. Download the Chromium browser binary
playwright install chromium

# 3. Run the automated scenario
python3 playwright_scenario.py
```

## AI Assistant [18:07:00]

**Chromium** is the open-source web browser engine that powers Google Chrome, Microsoft Edge, and Opera. 

You need it because Playwright is an E2E (End-to-End) visual testing tool:
1. Playwright cannot test your website's UI (forms, layouts, and JavaScript buttons) in a vacuum; it has to launch an actual web browser to render the HTML/CSS/JS.
2. It uses a custom, lightweight version of **Chromium** that Microsoft distributes so it can control it programmatically (opening pages, typing text, clicking links, and making assertions).
3. Without Chromium installed, Playwright will throw an error when running the script because it won't find a browser to run the tests on.

Since the `pip3` installation warned that the `playwright` command-line tool was placed in a directory outside your system PATH, you should install the Chromium browser binary by running the command through Python directly to avoid PATH issues:

```bash
python3 -m playwright install chromium
```

Once that completes, you can run the test script:
```bash
python3 playwright_scenario.py
```
