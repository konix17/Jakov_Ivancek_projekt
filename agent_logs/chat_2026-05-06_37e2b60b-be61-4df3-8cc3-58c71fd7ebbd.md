## Why you get this error

SQL Server is using an encrypted connection by default, but the server is presenting a certificate that Windows does not trust.

That happens when:
- SQL Express uses a self-signed certificate
- SSMS requires a trusted certificate chain
- the certificate is not from a known CA

## How to fix it

In SSMS connect dialog:
1. Click `Options >>`
2. Go to `Connection Properties`
3. Check `Trust server certificate`
4. Then connect again

If you use the app later, the same setting is `TrustServerCertificate=True` in the connection string.

## Summary

The error is not about your database itself; it is about the SSL certificate used by SQL Server. For a local development instance, trusting the server certificate is the normal fix.

## What is EF?

- **EF (Entity Framework)** is an ORM: Object-Relational Mapper.
- It lets you work with a SQL database using C# classes instead of SQL commands.

### Key pieces
- `DbContext` = the main EF class that represents the database session.
- `DbSet<T>` = a table of entities, like `DbSet<Hotel>` means “Hotels table”.
- Entities = your model classes, like `Hotel`, `Room`, `Reservation`.

### What EF does
- Creates database schema from your classes.
- Tracks changes to objects.
- Saves changes to the SQL database with `SaveChanges()`.
- Loads related data automatically using relationships.
- Helps you avoid writing raw SQL for most operations.

### Example
- `HotelDbContext` has `DbSet<Hotel> Hotels`.
- When controller asks for hotels, EF runs SQL behind the scenes and returns C# objects.

---

## What is Routing?

- Routing decides which controller and action handles a given URL.
- In ASP.NET MVC, routes map URL patterns to controller methods.

### Default routing
- Pattern: `{controller=Home}/{action=Index}/{id?}`
- Example:
  - `/Home/Index` → `HomeController.Index()`
  - `/Hotels/Details/5` → `HotelsController.Details(5)`

### Attribute routing
- You can define URLs directly on controllers/actions using `[Route("...")]`.
- Example:
  - `[Route("hoteli")]` on `HotelsController`
  - `[Route("{id:int}")]` on `Details(int id)`

### What routing does
- Converts a browser URL into a method call.
- Supports custom URLs like `/hoteli` instead of `/Hotels/Index`.
- Lets you require parameter types, e.g. `{id:int}` means only numeric IDs work.

## What are migrations?

- Migrations are EF’s way to update the database schema when your model changes.
- You run:
  - `dotnet ef migrations add Name`
  - `dotnet ef database update`

### What migrations do
- Generate SQL schema changes from your C# model.
- Apply those changes to the database.
- Keep the database in sync with your code.

---

## What is seeding?

- Seeding means inserting initial data automatically when the database is created.
- In EF, you use `modelBuilder.Entity<...>().HasData(...)`.

### Example
- Add sample hotels, rooms, guests, reservations, payments.
- After migration, the database already contains test data.

---

## What is the repository pattern?

- A repository is a class that handles data access.
- Example: `IHotelRepository` and `EfHotelRepository`.

### Why use it
- Keeps controllers clean.
- Lets you switch data source later (mock data, EF, API).
- Controller calls repository, repository uses EF.

---

## In your app, how it all works

1. Browser requests `/hoteli`
2. Routing finds `HotelsController.Index`
3. Controller asks repository for hotels
4. Repository uses `HotelDbContext` / EF to query the database
5. EF returns hotel objects
6. Controller returns a view with those objects
