# Deployment Guide

This guide explains how to deploy the Hotel Management Web Application to a Cloud Provider (Google Cloud, Azure, AWS) or a Virtual Machine (VM) using Docker.

## Prerequisites
- Docker installed on the target machine/VM.
- Docker Compose installed.

## Option 1: Running with Docker Compose (Recommended for VMs)

1. Clone or copy the project files to your Virtual Machine.
2. Navigate to the project root directory.
3. Run the following command to build and start the container:
   ```bash
   docker compose up -d --build
   ```
4. Access the web application in your browser at `http://<your-vm-ip>:8080`.

## Option 2: Deploying to Google Cloud Run (PaaS)

Google Cloud Run is a fully managed platform that automatically scales your stateless containers.

1. **Build and Tag the Docker image** using Google Artifact Registry:
   ```bash
   gcloud builds submit --tag gcr.io/<your-project-id>/hotel-mgt-web --project <your-project-id>
   ```
2. **Deploy to Cloud Run**:
   ```bash
   gcloud run deploy hotel-mgt-web \
     --image gcr.io/<your-project-id>/hotel-mgt-web \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated \
     --port 8080
   ```
3. Copy the URL generated by Cloud Run to access the application.

## Option 3: Deploying to Azure App Service (PaaS)

Azure App Service allows you to run containers directly from a registry.

1. **Create an Azure Container Registry (ACR)** and push the image:
   ```bash
   az acr create --resource-group myResourceGroup --name myRegistry --sku Basic
   az acr login --name myRegistry
   docker build -t myregistry.azurecr.io/hotel-mgt-web -f Hotel-Mgt.Web/Dockerfile .
   docker push myregistry.azurecr.io/hotel-mgt-web
   ```
2. **Create a Web App for Containers**:
   ```bash
   az appservice plan create --name myAppServicePlan --resource-group myResourceGroup --sku B1 --is-linux
   az webapp create --resource-group myResourceGroup --plan myAppServicePlan --name hotel-manager-app --deployment-container-image-name myregistry.azurecr.io/hotel-mgt-web:latest
   ```
3. Configure the app settings to expose port `8080` if needed, and access it via `https://hotel-manager-app.azurewebsites.net`.
